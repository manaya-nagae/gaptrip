# GapTrip MVP 設計書

## 1. プロダクト概要

旅行の「妄想」(行きたい場所)と「現実」(行った場所)のギャップを記録・可視化するWebアプリ。日本地図で都道府県の制覇率を確認しながら、スポットごとに期待度と満足度を記録し、そのギャップを楽しむ。

## 2. ターゲットユーザー

- 旅行好きの個人
- 行きたい場所をリスト管理し、実際に行った後の振り返りを楽しみたい人

## 3. 解決したい課題

- 行きたい場所が散在していて管理できない
- 旅行後に「期待と実際どうだったか」を振り返る手段がない
- 自分がどれだけ日本を巡ったか、一目で把握できない

## 4. 提供価値

| 価値 | 説明 |
|------|------|
| 可視化 | 日本地図で制覇率(XX/47)を一目で把握 |
| 記録 | 行きたい場所(WANT)と行った場所(WENT)を蓄積 |
| ギャップの楽しさ | 期待度 vs 満足度の差を「ギャップ」として振り返りを楽しくする |
| 導線 | 地図 → 都道府県 → スポット一覧という直感的なナビゲーション |

## 5. 技術スタック

- **フロントエンド**: Next.js 15 (App Router) / TypeScript / Tailwind CSS
- **データ取得**: Server Components + Prisma（DBへの直接アクセス）
- **更新処理**: Server Actions
- **ファイルアップロード**: Route Handler
- **ORM**: Prisma
- **DB**: PostgreSQL 16
- **認証**: NextAuth.js v5 (Credentials Provider / メール+パスワード)
- **開発環境**: Docker Compose
- **デプロイ**: MVP期間はローカル開発のみ

## 6. MVP要件一覧

### 6.1 含めるもの

- ユーザー認証（メール+パスワード、登録・ログイン）
- トップ画面（日本地図 + 制覇率表示）
- スポット一覧（都道府県・ステータスでフィルタ、ソート）
- スポット登録（WANT）
- スポット詳細表示
- スポット編集
- スポット削除（関連画像も削除）
- 現実記録（WANT → WENT 切り替え + 満足度・感想・訪問日）
- 画像アップロード（WENT記録時のみ、1スポット1枚、ローカル保存）
- ギャップスコアの自動計算・表示
- 固定カテゴリ（食事・観光・宿泊・アクティビティ・買い物・その他）
- プロフィール画面（名前表示 + ログアウト）

### 6.2 含めないもの（将来の拡張候補）

- パスワードリセット
- キーワード検索
- ページネーション / 無限スクロール
- 複数画像アップロード
- クラウドストレージ（S3等）への画像保存
- ソーシャル機能（他ユーザーのスポット閲覧・共有）
- プロフィール編集（アイコン、名前変更等）
- カテゴリのユーザー定義

## 8. 主要ユーザーフロー

### 8.1 新規登録 → 初回利用

```
ユーザー登録 → ログイン → マップ（全県グレー、制覇率0%）
→ ボトムナビで一覧へ → 「+」ボタン → スポット登録（WANT）→ 一覧に表示される
```

### 8.2 妄想を登録する（WANTフロー）

```
一覧画面 → 「+」ボタン → スポット登録画面
→ スポット名・都道府県・カテゴリ・期待度★・メモを入力
→ 保存 → 一覧に戻る（WANTバッジ付きで表示）
```

### 8.3 現実を記録する（WANT → WENTフロー）

```
一覧画面 → スポットカードタップ → 詳細画面
→ 「行ってきた！」ボタン → 現実記録画面
→ 満足度★・感想・画像・訪問日を入力 → 保存
→ 詳細画面に戻る（WENTバッジ、ギャップスコア表示）
```

### 8.4 地図から探す

```
マップ画面 → 都道府県タップ → 一覧画面（その都道府県でフィルタ済み）
→ スポットカードタップ → 詳細画面
```

## 9. データモデル

### 9.1 User

| カラム | 型 | 必須 | 説明 |
|--------|------|------|------|
| id | String (cuid) | PK | |
| email | String | Yes, Unique | メールアドレス |
| password | String | Yes | ハッシュ済みパスワード |
| name | String | No | 表示名 |
| createdAt | DateTime | Yes (default: now) | 作成日時 |
| updatedAt | DateTime | Yes (auto) | 更新日時 |

### 9.2 Spot

| カラム | 型 | 必須 | 説明 |
|--------|------|------|------|
| id | String (cuid) | PK | |
| userId | String | FK → User | 所有ユーザー |
| name | String | Yes | スポット名 |
| prefectureCode | String | Yes | 都道府県コード（"01"〜"47"） |
| categoryId | String | FK → Category | カテゴリ |
| status | Enum (WANT/WENT) | Yes (default: WANT) | ステータス |
| expectation | Int (1-5) | Yes | 期待度 |
| satisfaction | Int (1-5) | No | 満足度（WENT記録時に入力） |
| memo | String | No | メモ（WANT時の期待・気になること） |
| review | String | No | 感想（WENT記録時の振り返り） |
| imagePath | String | No | 画像ファイルパス（WENT記録時のみ設定） |
| visitedAt | DateTime | No | 訪問日（WENT記録時に入力、任意） |
| createdAt | DateTime | Yes (default: now) | 作成日時 |
| updatedAt | DateTime | Yes (auto) | 更新日時 |

**ギャップスコア**: `satisfaction - expectation` で都度計算（-4 〜 +4）。カラムには持たない。

### 9.3 Category（シードデータ）

| カラム | 型 | 必須 | 説明 |
|--------|------|------|------|
| id | String (cuid) | PK | |
| name | String | Yes | カテゴリ名 |
| sortOrder | Int | Yes | 表示順 |

初期データ:

| name | sortOrder |
|------|-----------|
| 食事 | 1 |
| 観光 | 2 |
| 宿泊 | 3 |
| アクティビティ | 4 |
| 買い物 | 5 |
| その他 | 6 |

### 9.4 Prefecture（コード定数）

テーブルではなく、アプリケーションコード内の定数として管理する。

```typescript
// 例
const PREFECTURES = [
  { code: "01", name: "北海道", region: "北海道" },
  { code: "02", name: "青森県", region: "東北" },
  // ...
  { code: "47", name: "沖縄県", region: "九州・沖縄" },
] as const;
```

## 10. 画面詳細仕様

### 10.1 ログイン画面 (`/login`)

**入力項目:**
- メールアドレス（テキスト、必須）
- パスワード（パスワード、必須）

**アクション:**
- ログインボタン → 認証成功でマップ画面へ、失敗でエラーメッセージ表示
- 「アカウントを作成」リンク → ユーザー登録画面へ
- 認証済みユーザーがアクセスした場合はマップ画面へリダイレクト

### 10.2 ユーザー登録画面 (`/register`)

**入力項目:**
- 名前（テキスト、任意）
- メールアドレス（テキスト、必須）
- パスワード（パスワード、必須、8文字以上）

**アクション:**
- 登録ボタン → アカウント作成後、自動ログインしてマップ画面へ
- 「ログインはこちら」リンク → ログイン画面へ
- 認証済みユーザーがアクセスした場合はマップ画面へリダイレクト

**バリデーション:**
- メール形式チェック
- メール重複チェック
- パスワード8文字以上

### 10.3 マップ画面 (`/`) — ホーム

**表示要素:**
- 日本地図（SVG）
  - 訪問済み都道府県（WENTスポットが1件以上）: テーマカラーで塗りつぶし
  - 未訪問: グレー
- 制覇率: 「XX / 47（XX%）」

**操作:**
- 都道府県タップ → スポット一覧へ遷移（`/spots?prefecture=XX`）

**初期表示:**
- ログインユーザーの全Spotデータを取得し、都道府県ごとにWENTの有無を判定
- 制覇率 = WENTスポットが存在する都道府県数 / 47（ログインユーザーのデータのみ対象）

### 10.4 スポット一覧画面 (`/spots`)

**表示要素:**
- フィルタ
  - 都道府県プルダウン（初期値: すべて、マップからの遷移時は該当都道府県）
  - ステータスプルダウン（すべて / WANT / WENT）
- ソートプルダウン（更新日:新しい順 / 期待度:高い順 / 都道府県順）
- 検索件数（「全X件」）
- スポットカードリスト
- FAB「+」ボタン

**スポットカード表示項目:**
- サムネイル画像（画像がない場合はカテゴリに応じたプレースホルダー）
- スポット名
- 都道府県名
- カテゴリ名
- ステータスバッジ（WANT / WENT）
- スコア: WANT時は「期待度★X」、WENT時は「期待度★X → 満足度★X（ギャップ+Y）」

**操作:**
- フィルタ・ソートのプルダウン変更 → URLクエリパラメータを更新（`/spots?prefecture=13&status=WANT&sort=expectation`）→ Server Componentが `searchParams` を読んで再レンダリング
- スポットカードタップ → スポット詳細へ
- FAB「+」→ スポット登録へ

**0件時:**
- フィルタなしで0件: 「まだスポットがありません。+ボタンから登録してみましょう！」
- フィルタありで0件: 「条件に一致するスポットが見つかりません」

### 10.5 スポット登録画面 (`/spots/new`)

**入力項目:**
- スポット名（テキスト、必須、最大100文字）
- 都道府県（プルダウン、必須）
- カテゴリ（プルダウン、必須）
- 期待度（★1〜5、必須）
- メモ（テキストエリア、任意、最大1000文字）

> 画像はWENT記録時にのみ登録可能なため、この画面では入力しない。

**アクション:**
- 保存ボタン → Spot作成（status=WANT）→ スポット一覧へ戻る
- キャンセル / 戻る → スポット一覧へ

### 10.6 スポット詳細画面 (`/spots/[id]`)

**表示要素:**
- 画像（あれば表示）
- スポット名
- 都道府県名
- カテゴリ名
- ステータスバッジ（WANT / WENT）
- 期待度（★表示）
- メモ
- WENT時のみ追加表示:
  - 満足度（★表示）
  - ギャップスコア（+X / -X / ±0）
  - 感想
  - 訪問日
- 登録日・更新日

**アクション:**
- WANT時:
  - 「行ってきた！」ボタン → 現実記録画面へ
  - 編集ボタン → スポット編集画面へ
  - 削除ボタン → 確認ダイアログ → 削除 → 一覧へ
- WENT時:
  - 編集ボタン → スポット編集画面へ
  - 削除ボタン → 確認ダイアログ → 削除 → 一覧へ

### 10.7 スポット編集画面 (`/spots/[id]/edit`)

登録済みSpotの情報を修正する画面。WANT・WENT問わず利用する。

**入力項目（WANT / WENT 共通）:**
- スポット名（テキスト、必須、最大100文字、既存値プリセット）
- 都道府県（プルダウン、必須、既存値プリセット）
- カテゴリ（プルダウン、必須、既存値プリセット）
- 期待度（★1〜5、必須、既存値プリセット）
- メモ（テキストエリア、任意、既存値プリセット）

**入力項目（WENT時のみ追加）:**
- 満足度（★1〜5、必須、既存値プリセット）
- 感想（テキストエリア、任意、既存値プリセット）
- 訪問日（日付選択、任意、既存値プリセット）

> 画像はこの画面では変更できない。現実記録画面（/record）でのWENT記録時のみ登録可能。

**アクション:**
- 保存ボタン → 更新 → スポット詳細へ戻る
- キャンセル / 戻る → スポット詳細へ

### 10.8 現実記録画面 (`/spots/[id]/record`)

WANTスポットをWENTに切り替える初回記録専用画面。WENT済みのSpotからはこの画面にアクセスできない（詳細画面にリダイレクト）。

**表示要素（参考表示）:**
- スポット名（読み取り専用）
- 期待度（読み取り専用、★表示）

**入力項目:**
- 満足度（★1〜5、必須）
- 感想（テキストエリア、任意、最大1000文字）
- 画像（ファイル選択、任意、JPEG/PNG、最大5MB）
- 訪問日（日付選択、任意）

**アクション:**
- 保存ボタン → status を WENT に更新、満足度・感想・画像・訪問日を保存 → スポット詳細へ
- キャンセル / 戻る → スポット詳細へ

### 10.9 プロフィール画面 (`/profile`)

**表示要素:**
- ユーザー名（未設定の場合は「未設定」）
- メールアドレス

**アクション:**
- ログアウトボタン → セッション破棄 → ログイン画面へ

## 11. 共通UI仕様

### 11.1 ボトムナビゲーション

3タブ構成:
- マップ（ホームアイコン）→ `/`
- 一覧（リストアイコン）→ `/spots`
- プロフィール（ユーザーアイコン）→ `/profile`

現在のタブはハイライト表示。ログイン・登録画面では非表示。

### 11.2 ローディング

- データ取得中はスケルトンUIまたはスピナーを表示

### 11.3 エラーハンドリング

- APIエラー時はトースト通知でメッセージ表示
- ネットワークエラー時は「接続を確認してください」

### 11.4 レスポンシブ対応

- モバイルファースト（主にスマホでの使用を想定）
- PCでも問題なく使えるレイアウト

## 12. 非機能要件

| 項目 | 内容 |
|------|------|
| 認証 | NextAuth.js Credentials Provider。セッション管理はJWT |
| パスワード保存 | bcrypt でハッシュ化 |
| 画像保存 | `public/uploads/` にローカル保存。ファイル名はUUID。Docker環境では `uploads` ディレクトリを bind mount で永続化し、コンテナ再作成後も画像が失われないようにする |
| 画像制限 | JPEG/PNG、最大5MB、1スポット1枚。WENT記録時（/record）のみ登録可能。登録後の画像変更はMVPに含まない |
| 画像削除 | Spot削除時は関連画像ファイルも合わせて削除する |
| データアクセス制御 | 一覧取得・制覇率集計はログインユーザーのSpotのみ対象。詳細・更新・削除はuserIdを条件に絞り込む。他ユーザーのSpot IDへのアクセス時は404を返す |
| 論理削除 | 使用しない（物理削除） |
| 開発環境 | Docker Compose（app + PostgreSQL） |
| デプロイ | MVPではローカル開発のみ |

## 13. 制約事項

- 画像はローカルストレージ保存のため、デプロイ時にはストレージ戦略の見直しが必要
- Docker Compose運用では `public/uploads/` を bind mount で永続化する。Dockerfile に `RUN mkdir -p /app/public/uploads` が必要
- 画像はWENT記録時（/record）のみ登録可能。登録後の画像変更（差し替え・削除）はMVPに含まない
- パスワードリセット機能はMVPに含まない
- 1スポット1枚の画像制限（複数枚対応は将来）

## 14. 未決事項

実装レベルの未決事項は全て解決済み。

| # | 項目 | 決定 | 理由 |
|---|------|------|------|
| 1 | Docker uploads ディレクトリの永続化方式 | **bind mount** | 個人ローカル開発環境でのファイル確認・バックアップの利便性を優先 |
| 2 | 他ユーザーのSpot IDにアクセスした場合の返却方針 | **404** | リソースの存在を他ユーザーに知らせない（情報漏洩防止） |
| 3 | 同一スポット名の重複登録 | **許可する** | MVPではシンプルさを優先。ユーザーが自由に名前をつけられる方が自然 |
| 4 | 訪問日への未来日入力 | **許可する** | 「予定している旅行」の記録ユースケースが自然に存在するため |
